About me

It all started with a simple security question: why can a program be exploited? Chasing that answer led me to buffer overflows, reverse engineering, and eventually writing my own tooling in Nim and Python.

Over time I realized that to break into a system, I had to understand how it's built. So I started building things from scratch: an HTTP framework in C++, an MCP server for red teaming, a Discord bot, and an industrial monitoring system built with a team. On campus, I'm active in Ethical Hacking, AI/ML, Big Data, and Computer Networks labs.

Right now I'm learning Nim, Elixir, Assembly, Verilog, and Solidity in parallel, while exploring C#. Sounds like a lot, but each language teaches a different way of thinking, and that's exactly what I'm after.

Featured Security Tool · 2025

BOAR

Buffer Overflow Analyzer & Remediation

A tool for hunting buffer overflows in C/C++ code. Detects 60+ patterns from real-world CVEs, analyzes crashes, checks binary protections (Canary, NX, ASLR, PIE), and auto-generates PoC exploits via pwntools.
